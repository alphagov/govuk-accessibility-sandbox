Accordion with descriptions

What is it?

The pattern is a list of headings each with a description and some hidden content. When you select the heading or the description the hidden content is revealed. There is also a button to open and close all the content.

The pattern allows users to see a summary of available content which can then be expanded if a user want to read more.

Accessibility success criteria

  • It must be possible to focus on the accordion controls using the tab key.
  • The accordion controls must change in appearance when keyboard focus moves to it.
  • It must be possible to activate the accordion controls using the appropriate keys.
  • The accordion control must change in appearance when the mouse is hovered over it.
  • It must be possible to activate the accordion controls using a click.
  • The accordion control must be large enough to tap accurately with one finger.
  • The accordion control must change in appearance to indicate the touch-down state.
  • It must be possible to activate the accordion controls using voice commands.
  • The expanded/collapsed state of the accordion must be available in the code.
  • The relationship between the accordion controls and disclosed content must be available in the code.
  • It must be possible to focus on the accordion controls using standard screen reader button commands.
  • The accordion controls must have a visible text label.
  • The accordion text must use a default font size that makes reading comfortable.
  • The contrast between the accordion text label and its background must meet (or exceed) a ratio of 4.5:1.
  • The accordion control must change appearance when it is expanded/collapsed.
  • The markup must validate
  • The content must be available without javascript
  • The content must be in a logical order in the source code
