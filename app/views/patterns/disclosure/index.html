Disclosure

What is it?

The pattern for disclosing content is to have a control that toggles the display of additional information.

Accessibility success criteria

  • It must be possible to focus on the disclosure control using the tab key.
  • The disclosure control must change in appearance when keyboard focus moves to it.
  • It must be possible to activate the disclosure control using the appropriate keys (for a button this is the space and enter keys).
  • The disclosure control must change in appearance when the mouse is hovered over it.
  • It must be possible to activate the disclosure control using a click.
  • The disclosure control must be large enough to tap accurately with one finger.
  • The disclosure control must change in appearance to indicate the touch-down state.
  • It must be possible to activate the disclosure control using standard button voice commands.
  • The expanded/collapsed state of the disclosure widget must be available in the code.
  • The relationship between the disclosure control and disclosed content must be available in the code.
  • It must be possible to focus on the disclosure control using standard screen reader button commands.
  • The disclosure control must have a visible text label.
  • The disclosure control text label must use a default font size that makes reading comfortable.
  • The contrast between the disclosure control text label and its background must meet (or exceed) a ratio of 4.5:1.
  • The disclosure control must change appearance when it is expanded/collapsed.
  • The markup must validate
  • The content must be available without javascript
  • The content must be in a logical order in the source code
