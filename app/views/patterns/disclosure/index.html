Disclosure

The pattern for disclosing content is to have a button that toggles the display of additional information.

Accessibility success criteria

  • It must be possible to focus on the disclosure button using the tab key.
  • The disclosure button must change in appearance when keyboard focus moves to it.
  • It must be possible to activate the disclosure button using the space and enter keys.
  • The disclosure button must change in appearance when the mouse is hovered over it.
  • It must be possible to activate the disclosure button using a click.
  • The disclosure button must be large enough to tap accurately with one finger.
  • The disclosure button must change in appearance to indicate the touch-down state (see General #4).
  • It must be possible to activate the disclosure button using standard button voice commands.
  • The expanded/collapsed state of the disclosure widget must be available in the code.
  • The relationship between the disclosure button and disclosed content must be available in the code.
  • It must be possible to focus on the disclosure button using standard screen reader button commands.
  • The disclosure button must have a visible text label.
  • The disclosure button text label must use a default font size that makes reading comfortable.
  • The contrast between the disclosure button text label and its background must meet (or exceed) a ratio of 4.5:1.
  • The disclosure button must change appearance when it is expanded/collapsed.
