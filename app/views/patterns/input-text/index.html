Accessibility success criteria

  • It must be possible to place focus in the text input by clicking or tapping on the label and/or the field.
  • It must be possible to place focus in the text input using the keyboard (tab key).
  • It must be possible to place focus in the text input using the keyboard (screen reader shortcuts).
  • When the text input receives focus screen readers must announce the label automatically.
  • When focus is placed in the text input it must be possible to enter characters using the keyboard.
  • When focus is placed in the text input it must be possible to enter characters using voice commands.
  • It must be possible to identify the corresponding label for the text input when using screen magnification.
  • It must be possible to use the text input when JavaScript is unavailable.
  • It must be possible to use the text input when CSS is unavailable.
